# 登壇候補者リスト

## 登壇者選定の基準
優先度の高い順に選定基準を定義します。

### 必須条件
- [ ] イベントテーマに関連する専門性がある
- [ ] 参加者の学びに繋がる実践的な知見を持っている
- [ ] 開催日時に登壇可能である

### 優先条件（Nice to Have）
- [ ] SNSフォロワーが多い（集客力がある）
- [ ] 登壇実績があり、話が分かりやすい
- [ ] 業界での知名度・影響力がある
- [ ] 多様性（性別、バックグラウンド、企業規模）

## 候補者の探し方

### 1. SNS検索
- Twitter/Xで関連ハッシュタグ検索（#技術, #エンジニア, #プロダクトマネジメント等）
- 影響力のあるアカウントをフォロー・リストアップ

### 2. 過去の登壇実績
- connpass, Doorkeeper等で類似イベントを検索
- 過去の登壇者をリストアップ

### 3. 技術ブログ・記事
- Qiita, Zenn, note等で良質な記事を書いている人
- 企業の技術ブログの執筆者

### 4. コミュニティ
- SlackやDiscordのコミュニティで活発な人
- OSSコントリビューター

### 5. 紹介・推薦
- 運営メンバーや知人からの紹介
- 既に承諾した登壇者からの推薦
