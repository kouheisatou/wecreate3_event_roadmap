# SNS運用ガイドライン

## 基本方針

### 運用目的
1. **認知拡大**: イベントの存在を知ってもらう
2. **集客**: チケット申込に繋げる
3. **コミュニティ形成**: 参加者・登壇者との継続的な関係構築
4. **ブランディング**: イベントの世界観・価値観を伝える

### ターゲット
- **メイン**: [例：エンジニア（3〜5年目）、技術に関心の高い層]
- **サブ**: [例：プロダクトマネージャー、デザイナー、学生]

---

## 運用体制

### 役割分担
| 役割 | 担当者 | 責任範囲 |
|------|--------|---------|
| **SNS統括** | [名前] | 全体戦略、最終承認 |
| **投稿担当** | [名前] | 日々の投稿作成・予約 |
| **デザイン** | [名前] | 画像・動画制作 |
| **エンゲージメント** | [名前] | リプライ・リツイート対応 |
| **分析** | [名前] | 数値分析・レポート |

### 承認フロー
1. 担当者が投稿案を作成
2. SNS統括が確認・承認
3. 予約投稿 or 即時投稿

**緊急時（当日のリアルタイム投稿等）**: 担当者の裁量で即投稿OK

## プラットフォーム別運用方針

### Twitter / X
**優先度**: ⭐️⭐️⭐️⭐️⭐️（最重要）

#### 基本設定
- **アカウント名**: [イベント名 公式]
- **@ID**: @[event_name]
- **プロフィール**: 
  ```
  [イベント名] 公式アカウント🎉
  XX月XX日（X）開催 | [会場名]
  [キャッチコピー]
  申込: [短縮URL]
  ```
- **ヘッダー画像**: イベントビジュアル（1500×500px）
- **アイコン**: イベントロゴ（400×400px）

#### 投稿方針
- **投稿頻度**: 週3〜5回（告知期間中は毎日）
- **最適投稿時間**: 
  - 平日: 12:00〜13:00、20:00〜22:00
  - 休日: 10:00〜11:00、20:00〜21:00
- **文字数**: 140文字以内を推奨（読みやすさ重視）
- **画像**: 必ず1枚以上添付（エンゲージメント率2倍）
- **ハッシュタグ**: 1〜3個まで（#[イベント名] は必須）

#### エンゲージメント
- **リプライ**: 24時間以内に返信
- **リツイート**: 参加者・登壇者の投稿は積極的にRT
- **いいね**: 関連投稿にいいねして認知獲得

## トンマナ（トーン＆マナー）

### 文体
- **基本**: です・ます調
- **親しみやすさ**: 絵文字を適度に使用（🎉✨📢等）
- **固すぎない**: カジュアルだが丁寧

#### 良い例
```
🎉登壇者紹介です！

[名前]さんが登壇されます✨
テーマは「XXX」

お楽しみに！
```

#### 悪い例
```
登壇者の発表を行います。XX氏が登壇予定です。（固すぎる）
```

```
やばい登壇者キターーー！！！（カジュアルすぎる）
```

---

### 絵文字の使い方
- **OK**: 🎉✨📢🔥👏🎤📍🗓💡
- **NG**: 過度な絵文字の連続、意味不明な絵文字

#### 良い例
```
🗓 日時: XX月XX日
📍 会場: XX会議室
```

#### 悪い例
```
🎉🎉🎉🎉🎉 イベント開催 🎉🎉🎉🎉🎉（うるさい）
```

---

### ハッシュタグ
#### 必須ハッシュタグ
- `#[イベント名]`（公式タグ）

#### 推奨ハッシュタグ
- `#エンジニア`
- `#tech`
- `#勉強会`
- `#IT`
- `#プログラミング`

#### 注意点
- Twitter: 1〜3個まで（読みやすさ重視）
- Instagram: 10〜20個（リーチ拡大のため多めに）

## 投稿カテゴリ

### 1. 告知系
**目的**: イベント情報の周知、申込促進

**例**:
```
🎉[イベント名] 開催決定！

XX月XX日（X）XX:XX〜
テーマ: [テーマ]

申込はこちら👇
[URL]

#[イベント名] #エンジニア
```

---

### 2. 登壇者紹介
**目的**: 登壇者の魅力を伝え、集客に繋げる

**例**:
```
📢登壇者紹介

[名前]さん（@username）
[所属] [役職]

セッション:
「[タイトル]」

[一言紹介]

#[イベント名]
```

---

### 3. スポンサー紹介
**目的**: スポンサーへの感謝、協賛メリットの可視化

**例**:
```
🙏スポンサー紹介

[企業名]様（@username）
にゴールドスポンサーとして
ご協賛いただいております！

[企業の紹介文]

ありがとうございます✨

#[イベント名]
```

---

### 4. カウントダウン
**目的**: 直前の盛り上げ、リマインド

**例**:
```
⏰あと X 日！

[イベント名] まであと X 日🔥

まだ間に合います！
お申込みはこちら👇
[URL]

#[イベント名] #カウントダウン
```

---

### 5. 当日リアルタイム
**目的**: 臨場感の共有、ハッシュタグの活性化

**例**:
```
📸会場の様子

たくさんの方にお越しいただいています🙏
ハッシュタグ #[イベント名] で
実況をお願いします！

[写真]
```

---

### 6. お礼・事後報告
**目的**: 感謝の伝達、次回への布石

**例**:
```
🎉無事終了しました！

ご参加いただいた皆様、
登壇者の皆様、
スポンサーの皆様、
本当にありがとうございました🙏

開催レポートは後日公開します✨

#[イベント名]
```

## エンゲージメント対応

### リプライ対応
#### パターン #1: 質問
```
ユーザー: 「当日の持ち物は何ですか？」

運営: 「ご質問ありがとうございます！
特に持ち物は不要ですが、名刺をお持ちいただくと
交流がスムーズです✨」
```

#### パターン #2: 好意的なコメント
```
ユーザー: 「楽しみにしてます！」

運営: 「ありがとうございます！
当日お会いできるのを楽しみにしています🎉」
```

#### パターン #3: ネガティブなコメント
```
ユーザー: 「参加費が高いですね…」

運営: 「貴重なご意見ありがとうございます。
運営費（会場、ケータリング等）に充てさせていただいております。
ご理解いただけますと幸いです🙏」
```

---

### リツイート基準
#### 積極的にRT
- 参加者の申込報告
- 登壇者の告知協力
- スポンサーの協賛告知
- ポジティブな感想

#### RTしない
- 関係のない内容
- ネガティブな批判（リプライで対応）
- スパム・宣伝

## 禁止事項

### NG行動
- ❌ 炎上を煽る表現
- ❌ 他イベントの批判
- ❌ 政治・宗教・差別的な発言
- ❌ 誤情報の拡散
- ❌ 無断での個人情報の投稿
- ❌ 過度な営業・宣伝

### NG表現
- ❌ 「炎上覚悟で言いますが…」
- ❌ 「〇〇は終わった」
- ❌ 「〇〇じゃない人は来ないで」

---

## 炎上対応マニュアル

### Step 1: 事実確認
- 何が問題視されているか把握
- 投稿内容に誤りがないか確認

### Step 2: 初動対応
- **誤りがある場合**: 速やかに謝罪・訂正
- **誤解の場合**: 丁寧に説明

### Step 3: 報告・相談
- 運営チーム全体に報告
- 必要に応じて弁護士等に相談

### Step 4: 再発防止
- 投稿フローの見直し
- ダブルチェック体制の強化
