# 緊急連絡体制構築

## タスク概要
当日のトラブルや緊急事態に迅速に対応できるよう、スタッフ・関係者の連絡手段を事前に整備します。LINEグループを活用することで、リアルタイムなコミュニケーションを可能にします。

## 目的
- 当日の緊急時に即座に連絡が取れる体制を作る
- 全スタッフ・関係者の連絡先を一元管理する
- トラブル発生時のエスカレーションルートを明確化する

## 実施手順
1. **連絡先収集**（1時間）
   - 全スタッフの電話番号・LINE IDを収集
   - 登壇者の緊急連絡先を確認
   - 会場担当者の連絡先を確保
   - スポンサー担当者の連絡先を確保

2. **LINEグループ作成**（30分）
   - 運営メンバー用グループ作成
   - 登壇者用グループ作成（必要に応じて）
   - 緊急時のみ使用するグループと通常連絡用を分ける

3. **緊急連絡網作成**（30分）
   - 責任者から各担当への連絡フロー図を作成
   - トラブル種別ごとの連絡先を明記

## 成果物
- 緊急連絡網リスト
- LINEグループ
- エスカレーションフロー図

## コピー可能なテンプレート

```markdown
# 緊急連絡網

## 当日緊急連絡先

| 役割 | 氏名 | 電話番号 | LINE |
|------|------|----------|------|
| 統括責任者 | 〇〇 | 090-XXXX-XXXX | ✓ |
| 会場責任者 | 〇〇 | 090-XXXX-XXXX | ✓ |
| 登壇者担当 | 〇〇 | 090-XXXX-XXXX | ✓ |
| 広報担当 | 〇〇 | 090-XXXX-XXXX | ✓ |

## 外部関係者

| 組織 | 担当者 | 電話番号 | 備考 |
|------|--------|----------|------|
| 会場（〇〇ビル） | 〇〇さん | 03-XXXX-XXXX | 設備トラブル時 |
| 登壇者A | 〇〇さん | 090-XXXX-XXXX | 当日朝連絡 |
| ケータリング | 〇〇社 | 03-XXXX-XXXX | 配送・数量調整 |

## エスカレーションフロー

### 会場トラブル（機材故障、設備異常）
1. 現場スタッフが発見
2. → 会場責任者に報告
3. → 会場担当者・統括責任者に連絡
4. → 判断・対応指示

### 登壇者遅延・欠席
1. 登壇者担当が把握
2. → 統括責任者に即報告
3. → プログラム調整・代替案検討

### 参加者体調不良
1. 現場スタッフが対応
2. → 統括責任者に報告
3. → 必要に応じて救急対応
```

- [ ] 緊急連絡網をドキュメント化し、全員に共有済み
- [ ] エスカレーションフローが明確になっている

## Tips
- **当日朝に確認**: 当日朝にLINEグループで全員の参加を確認
- **電話番号も必須**: LINEだけでなく電話番号も必ず収集（通信障害に備える）
- **会場Wi-Fiに頼らない**: モバイル回線で連絡が取れることを前提に
- **登壇者の到着確認**: 開始1時間前に登壇者全員に連絡して到着状況を確認

## 所要時間目安
1時間（リスト作成30分 + グループ設定30分）
