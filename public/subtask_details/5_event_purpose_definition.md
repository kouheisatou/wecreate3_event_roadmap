# イベントの目的と目標を明確にする

## タスク概要

イベントコンセプト設計の核心となるタスクです。「誰に・何を・何のために」を明確に言語化することで、イベントの方向性が定まります。曖昧な表現を避け、具体的なターゲット層、提供する価値、達成したい目標を定義します。この作業を丁寧に行うことで、その後の登壇者選定、スポンサー営業、集客メッセージなど、すべての活動に一貫性が生まれます。

## 目的

- イベントの「誰に・何を・何のために」を明確に定義する
- ターゲット参加者が抱える課題と、イベントが提供する解決策を明確化する
- 測定可能な目標（KPI）を設定する
- 登壇者選定やスポンサー営業の判断軸を作る
- 集客メッセージの軸を定める

## 実施手順

1. **「誰に」を明確にする**（60分）
   - ターゲット参加者の属性を具体化（年齢、職業、関心領域、抱えている課題）
   - 「20代」だけでなく「AI時代のキャリア形成に悩む20代のビジネスパーソン」のように具体化
   - メインターゲットとサブターゲットを区別

2. **「何を」を明確にする**（60分）
   - イベントを通じて参加者に提供する価値・体験を定義
   - 情報提供、ネットワーキング、スキル獲得、インスピレーションなど
   - 他の類似イベントとの差別化ポイントを明確化

3. **「何のために」を明確にする**（60分）
   - 参加者がイベント後に得られる成果・変化を定義
   - 「イベントに参加したことで、〇〇ができるようになる」
   - 短期的な成果と長期的な影響を分けて考える

4. **測定可能な目標（KPI）を設定する**（30分）
   - 参加者数、満足度、SNS投稿数、次回参加意向など
   - 数値目標を明確にする

5. **目的・目標定義書を作成**（30分）
   - 上記の内容を1つの文書にまとめる
   - メンバー全員で確認し、合意を得る

## 成果物

- 目的・目標定義書（イベントコンセプトシート）

## コピー可能なテンプレート

```markdown
# [イベント名] 目的・目標定義書

**作成日**: YYYY/MM/DD  
**作成者**: [運営チーム名]  
**Version**: 1.0

---

## 1. イベント概要

| 項目 | 内容 |
|------|------|
| イベント名 | [イベント名] |
| テーマ | [テーマ] |
| 開催日時 | YYYY/MM/DD HH:MM-HH:MM |
| 会場 | [会場名・住所] / オンライン |
| 想定参加者数 | 〇〇名 |

---

## 2. 誰に（ターゲット参加者）

### メインターゲット

- **属性**: [年齢、職業、関心領域]
- **抱えている課題**: [具体的な悩みや課題]
- **イベント参加動機**: [なぜこのイベントに参加するのか]

**例**:
- 属性: 20代のビジネスパーソン（エンジニア、マーケター、企画職など）
- 課題: AI時代にどうキャリアを築けば良いか不安。具体的なロールモデルや事例が少ない
- 動機: 同世代のAI活用事例を知りたい、キャリアの方向性のヒントを得たい

### サブターゲット

- **属性**: [メインターゲット以外の想定参加者]
- **期待**: [サブターゲットがイベントから得たいもの]

**例**:
- 属性: AI関連のスタートアップ経営者、人事担当者
- 期待: 若手人材の採用、自社サービスの認知拡大

---

## 3. 何を（提供する価値・体験）

このイベントを通じて、参加者に以下の価値を提供します：

### 提供価値1: [例: 具体的なロールモデルの提示]

- **内容**: AI時代に活躍する20代の登壇者から、キャリア形成の具体例を学ぶ
- **実現方法**: 登壇者によるパネルディスカッション、Q&Aセッション

### 提供価値2: [例: 同世代とのネットワーキング]

- **内容**: 同じ関心を持つ同世代と出会い、情報交換できる場を提供
- **実現方法**: 休憩時間の交流、オンラインコミュニティへの招待

### 提供価値3: [例: 実践的な知識の獲得]

- **内容**: AIツールを活用したキャリア形成の具体的な手法を学ぶ
- **実現方法**: ワークショップ、実践事例の共有

---

## 4. 何のために（目指す成果・変化）

### 参加者が得られる成果

イベント参加後、参加者は以下の状態になることを目指します：

- **短期的成果**（イベント終了直後）
  - [ ] AI時代のキャリア形成について、具体的なイメージを持てている
  - [ ] 少なくとも3つの新しい視点や気づきを得ている
  - [ ] 同世代の参加者と繋がりができている

- **中期的成果**（1ヶ月後）
  - [ ] AIツールを実際に活用し始めている
  - [ ] キャリアについて具体的なアクションを起こしている
  - [ ] イベントで得た知識を仕事に活かしている

- **長期的成果**（半年〜1年後）
  - [ ] AI時代に適応したキャリアを築けている
  - [ ] イベントで出会った仲間と継続的に交流している
  - [ ] 次世代に知見を共有するようになっている

---

## 5. 測定可能な目標（KPI）

| KPI項目 | 目標値 | 測定方法 |
|--------|--------|---------|
| 参加者数 | 〇〇名以上 | 受付チェックイン |
| 満足度 | 80%以上（5点満点で4点以上） | アンケート |
| SNS投稿数 | 〇〇件以上（ハッシュタグ） | SNS分析ツール |
| 次回参加意向 | 70%以上 | アンケート |
| スポンサー満足度 | 80%以上 | スポンサーアンケート |
| オンラインコミュニティ参加率 | 30%以上 | Discord参加者数 |

---

## 6. 他イベントとの差別化ポイント

このイベントが他の類似イベントと異なる点：

- **差別化1**: [例: 20代に特化したコンテンツ]
  - 一般的なAIイベントではなく、20代のキャリア形成に焦点を絞る

- **差別化2**: [例: 実践者の生の声]
  - 理論ではなく、実際にAIを活用してキャリアを築いている人の具体例を共有

- **差別化3**: [例: 継続的なコミュニティ]
  - イベント単発ではなく、Discord等で継続的に繋がれるコミュニティを提供

---

## 7. 判断軸（この目的に照らした意思決定基準）

運営中に判断に迷った際は、以下の問いに立ち返ります：

- この選択は、ターゲット参加者に価値を提供するか？
- この選択は、「誰に・何を・何のために」に合致するか？
- この選択は、KPI達成に貢献するか？

---

**承認者**: [統括者名]  
**承認日**: YYYY/MM/DD  
**最終更新日**: YYYY/MM/DD
```

## チェックリスト

- [ ] 「誰に」が具体的に定義されている（年齢、職業、抱えている課題など）
- [ ] 「何を」が明確になっている（提供する価値・体験）
- [ ] 「何のために」が言語化されている（参加者が得られる成果）
- [ ] 測定可能な目標（KPI）が設定されている
- [ ] 他イベントとの差別化ポイントが明確になっている
- [ ] メンバー全員が内容に合意している
- [ ] 文書を全メンバーに共有している

## Tips

- **曖昧な表現を避ける**: 「学び」「繋がり」「成長」などの抽象的な言葉ではなく、具体的に記載
- **ペルソナを作る**: 実在する友人・知人をイメージすると、ターゲットが具体化しやすい
- **競合イベントをリサーチ**: 類似イベントを調べ、差別化ポイントを明確にする
- **数値目標を設定**: 曖昧な目標ではなく、測定可能な数値目標を設定する
- **全員でディスカッション**: 統括者一人で決めるのではなく、メンバー全員で議論する
- **定期的に見直す**: 運営中に目的を見失わないよう、定期的にこの文書を見返す
- **参加者目線で考える**: 運営側の都合ではなく、参加者にとっての価値を中心に考える

## 所要時間目安

3時間（ターゲット定義60分 + 提供価値定義60分 + 成果定義60分 + KPI設定30分 + 文書作成30分）
