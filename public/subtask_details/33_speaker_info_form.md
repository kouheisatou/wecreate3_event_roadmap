# 登壇者情報回収フォーム

## タスク概要
登壇者から最終的なタイトル・概要・プロフィール・写真を回収します。これらの情報は、イベントページ、SNS告知、パンフレット等で使用します。

## 目的
- 登壇者情報を一元管理する
- 告知・制作物に使用する情報を収集する
- 登壇内容を最終確定する

## 実施手順
1. **Googleフォーム作成**
2. **登壇者にフォームURL送付**
3. **回答の確認・リマインド**
4. **情報の整理・共有**

## 成果物
- 登壇者情報一覧（スプレッドシート）

- **例**:

#### プロフィール（300字程度）

- **用途**: 詳細な紹介が必要な場合（Webサイト等）

#### プロフィール写真（顔写真）*

### 5. 登壇内容

#### 登壇形式*
- **形式**: 選択
- **選択肢**:
  - Keynote（基調講演）
  - Panel Discussion（パネルディスカッション）
  - Lightning Talk（ライトニングトーク）

#### セッションタイトル*
- **形式**: テキスト
- **例**: 
  - 「Web3が切り開く未来の金融システム」
  - 「学生起業家が語る、Web3で起業するということ」
  - 「AIとWeb3の融合がもたらす新しい可能性」

#### セッション概要（200字程度）*
- **形式**: 長文テキスト
- **用途**: イベントページ、SNS告知
- **例**: 
  ```
  Web3技術がどのように金融システムを変革しているのか、具体的な事例を交えて解説します。
  ブロックチェーン、DeFi、NFTなどの技術が、従来の金融システムのどのような課題を解決するのか、
  また今後の展望についてもお話しします。
  Web3に興味はあるが、具体的にどう社会に役立つのか分からないという方にも分かりやすく説明します。
  ```

#### 登壇で伝えたいこと（箇条書き3〜5個）*
- **形式**: 長文テキスト
- **例**: 
  ```
  - Web3技術の基礎知識
  - DeFiが従来の金融システムを変革する理由
  - 実際のDeFiプロジェクトの事例紹介
  - Web3領域でのキャリアの可能性
  - 学生が今からできるWeb3の学び方
  ```

## チェックリスト

- [ ] 提出期限が明確に設定されている
- [ ] 回答状況が管理されている
- [ ] 未回収者にリマインドメールが送られている
- [ ] 回収した情報が制作チームに共有されている

## Tips
- **Googleフォーム活用**: 手動で収集するより、Googleフォームで一元管理すると効率的
- **提出期限は余裕を持って**: 制作物入稿の2週間前には回収完了したい
- **リマインドは丁寧に**: 忙しくて忘れているだけなので、丁寧にリマインドする
- **画像サイズ指定**: プロフィール写真のサイズを指定しておくと、後で加工する手間が省ける
- **撮影可否の確認**: 必ず事前に撮影・録画の可否を確認しておく
- **複数回リマインド**: 1回で集まらないことが多いので、2〜3回リマインドする覚悟で
